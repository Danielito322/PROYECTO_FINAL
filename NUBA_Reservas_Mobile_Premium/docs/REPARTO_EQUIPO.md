## Reglas de trabajo
1. Crear una rama personal antes de cambiar código.
2. Probar con Live Server antes de subir.
3. No borrar archivos de otros módulos.
4. Avisar en el grupo antes de editar `data.js`, `app-shell.js` o CSS global.
5. Hacer commits pequeños y con mensajes claros.

# Aplicación de principios de interacción

1. **Visibilidad del estado:** chips, estados de reserva, toast y resumen de pago.
2. **Correspondencia con el mundo real:** categorías, calendario semanal y horarios claros.
3. **Control del usuario:** reprogramación, cancelación simulada y cambio de rol.
4. **Consistencia:** mismos radios, tamaños, jerarquías, botones y navegación inferior.
5. **Prevención de errores:** no permite continuar sin horario y diferencia horas ocupadas.
6. **Reconocimiento sobre memoria:** precios, extras y detalle siempre visibles antes del pago.
7. **Flexibilidad:** filtros, búsqueda, mapa y tres roles.
8. **Diseño minimalista:** modo oscuro, espacios amplios, degradados suaves y textos breves.
9. **Recuperación de errores:** mensajes de retroalimentación y mapa alternativo si la API falla.
10. **Ayuda contextual:** textos breves en cada paso, QR, tienda opcional y paneles.
